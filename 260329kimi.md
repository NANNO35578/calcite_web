# 文件树 Bug 修复记录

日期：2026-03-29

## 修复问题

### 问题1：文件树无法显示子文件夹

**问题描述**：用户登录后主界面左侧栏的文件树只显示根目录下文件夹以及根目录下笔记，不能显示子文件夹。

**根本原因**：
1. `CaretRight` 图标未导入导致组件警告
2. 递归组件的 `folders` 传递逻辑错误 - 递归调用时传递了空的 `folders`，但子组件依赖它渲染

**修复内容**（`calcite-web/src/components/FileTree.vue`）：

#### 1. 导入缺失的 `CaretRight` 图标（第 88-94 行）

```javascript
import {
  Folder,
  Document,
  MoreFilled,
  CaretBottom,
  CaretRight  // 新增
} from '@element-plus/icons-vue'
```

#### 2. 修改递归渲染逻辑

**模板部分**（第 4-7 行）：
```vue
<div
  v-for="folder in displayFolders"  <!-- 改为 displayFolders -->
  :key="folder.id"
  class="tree-node"
>
```

**递归调用部分**（移除 `:folders="childFolders"`）：
```vue
<FileTree
  :all-folders="allFolders"
  <!-- 删除 :folders="childFolders" -->
  :notes="notes"
  :parent-folder-id="folder.id"
  ...
/>
```

#### 3. 添加 `displayFolders` 计算属性（第 142-149 行）

```javascript
// 当前节点展示的文件夹（根级使用传入的folders，子级使用allFolders计算）
const displayFolders = computed(() => {
  if (props.isRoot) {
    return props.folders
  }
  // 子级从 allFolders 中根据 parentFolderId 筛选
  return props.allFolders.filter(f => f.parent_id === props.parentFolderId)
})
```

#### 4. 更新 `isEmpty` 计算属性

```javascript
const isEmpty = computed(() => {
  return displayFolders.value.length === 0 && currentLevelNotes.value.length === 0
})
```

**修复原理**：
- 根级调用时传入 `folders`（根文件夹列表），使用 `displayFolders` 渲染
- 递归调用时不传 `folders`，子组件根据 `parentFolderId` 从 `allFolders` 中筛选子文件夹
- 这样可以确保各级子文件夹都能正确显示

### 问题2：新建文件夹缺少父文件夹选项

**修复内容**：

#### 1. 文件夹对话框添加父文件夹选择器（`Home.vue` 第 264-283 行）

```vue
<el-form-item v-if="!editingFolder" label="父文件夹">
  <el-select v-model="folderForm.parentId" placeholder="选择父文件夹（默认根目录）" style="width: 100%">
    <el-option label="根目录" :value="0" />
    <el-option
      v-for="folder in allFolders"
      :key="folder.id"
      :label="getFolderPath(folder)"
      :value="folder.id"
    />
  </el-select>
</el-form-item>
```

#### 2. 默认父文件夹逻辑（`Home.vue` 第 737-745 行）

```javascript
const handleCreateFolder = (parentId) => {
  editingFolder.value = null
  // 使用当前选中的文件夹作为默认父文件夹，或使用明确传入的 parentId
  const effectiveParentId = (parentId !== undefined && parentId !== null)
    ? Number(parentId)
    : (selectedFolderId.value ?? 0)
  folderForm.value = { name: '', parentId: effectiveParentId }
  folderDialogVisible.value = true
}
```

**功能说明**：
- 新建文件夹时显示"父文件夹"选择器
- 默认逻辑：
  - 若 👈 指向了某个文件夹（`selectedFolderId` 有值），则默认选中该文件夹作为父文件夹
  - 若未指向任何文件夹，则默认为根目录（0）
- 可通过下拉框手动选择其他父文件夹
- 编辑文件夹时不显示此选项
