# Calcite Web 修复文档 - 工具栏按钮失效

**日期**: 2026-04-12  
**版本**: Hotfix v5.0  
**问题**: 中央顶部工具栏按钮点击无反应

---

## 问题描述

中央顶部工具栏的左右侧栏切换按钮点击后没有作用。

---

## 根因分析

按钮被包裹在 `el-tooltip` 组件中，点击事件可能被 tooltip 拦截或阻止冒泡，导致 `@click` 事件无法正常触发。

---

## 修复方案

移除 `el-tooltip` 包裹，改用原生的 `title` 属性实现悬浮提示。

### 修复前

```vue
<el-tooltip content="文件夹" placement="bottom">
  <el-button
    :type="leftCollapsed ? 'primary' : 'default'"
    @click="$emit('toggle-left')"
    ...
  />
</el-tooltip>
```

### 修复后

```vue
<el-button
  :type="leftCollapsed ? 'primary' : 'default'"
  @click.stop="$emit('toggle-left')"
  :title="leftCollapsed ? '展开文件夹' : '收起文件夹'"
  ...
/>
```

---

## 关键改动

| 项目 | 修复前 | 修复后 |
|------|--------|--------|
| 悬浮提示 | `el-tooltip` 组件 | `title` 属性 |
| 点击事件 | `@click` | `@click.stop` |
| 组件层级 | 2层 (tooltip+button) | 1层 (button) |

---

## 文件修改

### CenterToolbar.vue

- 移除 `el-tooltip` 包裹
- 添加 `@click.stop` 防止事件冒泡问题
- 使用 `:title` 属性替代 tooltip 的 `content`
