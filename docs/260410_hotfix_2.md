## 问题描述

### Bug: 侧边栏折叠后无法重新打开

**现象**: 
- 当用户点击侧边栏的折叠按钮后，左右侧边栏会收起
- 折叠后，原来的折叠按钮随着侧边栏一起隐藏
- 用户**没有任何 UI 按钮**可以重新展开侧边栏
- 刷新页面后侧边栏仍然保持折叠状态（localStorage 持久化）
- 用户陷入"死胡同"，无法使用文件夹和标签功能

**影响范围**: 所有用户界面

---

## 解决方案

### 设计思路

将侧边栏控制从**两侧栏边界处**移至**中间栏顶部工具栏**：
1. **删除**左右栏边界处的 `div.collapse-btn`
2. **保留**中间栏顶部工具栏的两个 toggle 按钮
3. **统一**由工具栏控制左右侧栏的展开/收起

**新的交互方式**:
- 栏展开时：按钮显示默认样式，文字为"收起"，点击收起侧边栏
- 栏折叠时：按钮显示主色样式，文字为"文件夹"/"标签"，点击展开侧边栏

## 设计决策

### 为什么选择统一工具栏控制？

**优点**:
1. **一致性** - 所有操作在同一个位置，用户学习成本低
2. **可靠性** - 按钮始终可见，不会出现"死胡同"
3. **简洁性** - 界面更干净，减少冗余按钮
4. **可维护性** - 代码更简单，逻辑集中

### 为什么不保留栏边界按钮？

**原因**:
1. **Bug 风险** - 栏折叠后按钮消失，导致无法展开
2. **冗余操作** - 同一功能有多个入口，造成困惑
3. **界面杂乱** - 边界按钮破坏视觉边界
4. **VS Code 模式** - 主流 IDE 均采用顶部工具栏控制
