# Calcite Web 修复文档 - 面板折叠优化

**日期**: 2026-04-12  
**版本**: Hotfix v4.0  
**问题**: 点击工具栏按钮后，面板内容不显示但面板区域还占据空间

---

## 问题描述

使用 `v-show` 在内部元素上控制显示/隐藏时：
- 面板内容隐藏了
- 但 `el-splitter-panel` 本身仍然占据空间
- 导致中间栏没有正确扩展填满

---

## 解决方案

使用 `el-splitter-panel` 的 `collapsible` 属性来实现真正的面板折叠。

### 修复前

```vue
<el-splitter-panel size="25%">
  <div v-show="!leftCollapsed" class="panel-content">
    <LeftSidebar />
  </div>
</el-splitter-panel>
```

### 修复后

```vue
<el-splitter-panel 
  size="25%" 
  :min="200" 
  :max="400"
  collapsible
  :collapsed="leftCollapsed"
  @collapse="leftCollapsed = true"
  @expand="leftCollapsed = false"
>
  <LeftSidebar />
</el-splitter-panel>
```

---

## 属性说明

| 属性 | 类型 | 说明 |
|------|------|------|
| `collapsible` | Boolean | 启用折叠功能 |
| `collapsed` | Boolean | 控制折叠状态 |
| `@collapse` | Event | 折叠时触发 |
| `@expand` | Event | 展开时触发 |

## 文件修改

### Home.vue

#### 左栏配置
```vue
<el-splitter-panel 
  size="25%" 
  :min="200" 
  :max="400"
  collapsible
  :collapsed="leftCollapsed"
  @collapse="leftCollapsed = true"
  @expand="leftCollapsed = false"
>
```

#### 右栏配置
```vue
<el-splitter-panel 
  size="25%" 
  :min="200" 
  :max="400"
  collapsible
  :collapsed="rightCollapsed"
  @collapse="rightCollapsed = true"
  @expand="rightCollapsed = false"
>
```

---

## 交互流程

1. 用户点击中间工具栏"文件夹"按钮
2. `leftCollapsed` 状态改变
3. `el-splitter-panel` 检测到 `:collapsed="true"`
4. 面板自动折叠，宽度变为 0（或最小折叠宽度）
5. 中间栏自动扩展填满剩余空间
